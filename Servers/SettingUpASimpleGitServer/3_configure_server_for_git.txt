We can configure our server for git as follows:

First, we enable our apache2 modules listed below, using a2enmod (apache 2 enable modules):
$ sudo a2enmod env cgi alias rewrite

We need to also create a folder to keep all our git repositories.


The next step is to create a new apache website. For this, we need to create a configuration file, which we call
git.conf, which we put under the configuration files at etc/apache2/sites-available/

$ sudo vim /etc/apache2/sites-available/git.conf

We must enter the following configuration in this file. Then, disable the default configuration as follows, and
then enable the git website as follows:

$ sudo a2dissite 000-default.conf
$ sudo a2ensite git.conf		# I think we should repeat this command everytime we change the .conf file.
$ sudo systemctl restart apache2

<VirtualHost *:80>
ServerAdmin webmaster@localhost
 
SetEnv GIT_PROJECT_ROOT PATH_TO_FOLDER
SetEnv GIT_HTTP_EXPORT_ALL
ScriptAlias /git/ /usr/lib/git-core/git-http-backend/
 
Alias /git PATH_TO_FOLDER
 
<Directory /usr/lib/git-core>
Options +ExecCGI -MultiViews +SymLinksIfOwnerMatch
AllowOverride None
Require all granted
</Directory>
 
DocumentRoot /var/www/html
 
<Directory /var/www>
Options Indexes FollowSymLinks MultiViews
AllowOverride None
Require all granted
</Directory>
 
 
ErrorLog ${APACHE_LOG_DIR}/error.log
LogLevel warn
CustomLog ${APACHE_LOG_DIR}/access.log combined
</VirtualHost>
